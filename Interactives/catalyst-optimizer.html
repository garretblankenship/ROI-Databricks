Meet the Catalyst Optimizer

When you call an action, Spark doesn't run your code directly. It hands your query to Catalyst — a query optimizer that parses, analyzes, rewrites, and plans it through four phases before a single task starts. Watch the transformation happen.

Your code is just a tree

Catalyst sees your DataFrame code as a tree of operators: a project on top of a filter on top of another filter on top of a relation.

Click Next Phase below to watch the tree transform through all four phases of optimization.

Inspect any plan yourself: Call df.explain(mode="extended") on any DataFrame to print all four plans. In the Spark UI, the SQL/DataFrame tab shows the physical plan as a visual DAG, with WholeStageCodegen markers showing which operators got fused into a single function. Reading these plans is the single best skill for tuning Spark.
