Beyond Catalyst: Photon and Predictive I/O

Catalyst gives you a smart plan. Databricks then adds two layers of acceleration on top: Photon, a native C++ engine that runs the plan with vectorized CPU instructions, and Predictive I/O, which uses ML to read only the bytes you need — and skip the rewrites you don't.

SCENARIO · 01
One row at a time vs eight at a time

Spark's classic engine generates Java bytecode that processes one row per CPU cycle. Each value goes through the JVM's interpreter and type-checking machinery.

Photon throws all that out. It's native C++ that uses SIMD (Single Instruction, Multiple Data) — CPU instructions that operate on 8 values at once. Same chip, 8× the work per cycle.

Click Start below to watch the same dataset stream through both engines.

The key idea Modern CPUs have always had SIMD instructions (SSE, AVX, AVX-512). JVM Spark mostly can't use them. Photon was built specifically to feed the CPU what it's actually fast at.

Why this matters: Photon and Predictive I/O are the two biggest reasons Databricks queries finish faster than open-source Spark on the same data. Photon doesn't change your code or your plan — it just runs the execution layer in C++ with vectorized CPU instructions. Predictive I/O doesn't add indexes or background jobs — it uses ML to read only the bytes a selective query needs, and replaces full-file rewrites with tiny bitmap deletion vectors. Both are on by default in modern Databricks SQL warehouses and Photon-enabled clusters.
